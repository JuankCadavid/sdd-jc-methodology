# v0.2.0 - Packaged SDD lifecycle and installer

## Included

- New SDD lifecycle commands:
  - `sdd-propose`
  - `sdd-archive`
- Updated command guidance for:
  - `sdd-specify`
  - `sdd-execute`
  - `sdd-test`
  - `sdd-validate`
- Node package metadata for npm/pnpm distribution
- `sdd-jc` CLI with:
  - `install`
  - `update`
  - `list`
  - `doctor`
- Installer support for:
  - Claude via `~/.claude`
  - OpenCode via `~/.config/opencode`
  - both tools with `--tool both`
- MIT license for public package distribution

## Highlights

- Adds a reviewable proposal step before full specification.
- Adds an archive step for completed SDD work.
- Documents Lite, Standard, and Full documentation depth.
- Adds requirement delta preview guidance using `ADDED`, `MODIFIED`, and `REMOVED` sections.
- Makes installation repeatable through `pnpm dlx sdd-jc-methodology install` after publication.
- Keeps installer behavior safe by skipping existing files unless `--force` is used.

## Install Examples

```bash
pnpm dlx sdd-jc-methodology install
pnpm dlx sdd-jc-methodology install --tool opencode
pnpm dlx sdd-jc-methodology install --tool both
```

From a local checkout:

```bash
node bin/sdd-jc.js install --tool both
```

## Verification Performed

- `node bin/sdd-jc.js list`
- `node bin/sdd-jc.js install --tool both --force` into temporary Claude and OpenCode targets
- `node bin/sdd-jc.js doctor --tool both` against temporary targets
- `node bin/sdd-jc.js install --tool opencode --dry-run`
- invalid `--tool both --target` handling
- `npm pack --dry-run`
- `git diff --check`

## Release Notes

- Package is licensed under MIT for public npm publishing.
- Dotfiles are intentionally excluded from the npm package.
- OpenCode users should restart OpenCode after install or update.
