# v0.2.0 - Packaged AKILI-SPECS lifecycle and installer

## Included

- New AKILI-SPECS lifecycle commands:
  - `akili-propose`
  - `akili-archive`
- Updated command guidance for:
  - `akili-specify`
  - `akili-execute`
  - `akili-test`
  - `akili-validate`
- Node package metadata for npm/pnpm distribution
- `akili` CLI with:
  - `install`
  - `update`
  - `list`
  - `doctor`
- Installer support for:
  - Claude via `~/.claude`
  - OpenCode via `~/.config/opencode`
  - both tools with `--tool both`
- MIT license for public package distribution

## Highlights

- Adds a reviewable proposal step before full specification.
- Adds an archive step for completed AKILI-SPECS work.
- Documents Lite, Standard, and Full documentation depth.
- Adds requirement delta preview guidance using `ADDED`, `MODIFIED`, and `REMOVED` sections.
- Makes installation repeatable through `pnpm dlx akili-specs install` after publication.
- Keeps installer behavior safe by skipping existing files unless `--force` is used.

## Install Examples

```bash
pnpm dlx akili-specs install
pnpm dlx akili-specs install --tool opencode
pnpm dlx akili-specs install --tool both
```

From a local checkout:

```bash
node bin/akili.js install --tool both
```

## Verification Performed

- `node bin/akili.js list`
- `node bin/akili.js install --tool both --force` into temporary Claude and OpenCode targets
- `node bin/akili.js doctor --tool both` against temporary targets
- `node bin/akili.js install --tool opencode --dry-run`
- invalid `--tool both --target` handling
- `npm pack --dry-run`
- `git diff --check`

## Release Notes

- Package is licensed under MIT for public npm publishing.
- Dotfiles are intentionally excluded from the npm package.
- OpenCode users should restart OpenCode after install or update.
